# SDK 55 Migration Deviations and Evidence Gaps

Date: 2026-07-16

## Current Acceptance Status

The migration implementation is code-vetted, but the migration is not release-accepted. The static typecheck gate remains unmet, iOS verification is pending, and required device and production-like behavior has not been exercised. No item below waives the acceptance criteria in [migration-plan.md](migration-plan.md).

| ID | Status | Deviation or evidence gap | Reason | Impact | Closure action |
| --- | --- | --- | --- | --- | --- |
| DEV-01 | Documented limitation | The SDK 54 static baseline was reconstructed from immutable commit `64dfcb4` after the migration; no SDK 54 native build or device seed was captured | Initial `node_modules` was absent, so the historical revision was installed and checked later | Static comparison is reproducible, but native and in-place upgrade claims cannot be made | If upgrade assurance is required, build SDK 54, seed the specified state on devices, then install SDK 55 over it and run Gates C through F |
| DEV-02 | Accepted version deviation | The plan expected React Native 0.83.10; Expo 55.0.28 selected and validated React Native 0.83.6 | Expo's compatible dependency set is authoritative | The documented estimate differs from the resolved version; overriding it could create an unsupported dependency set | Keep 0.83.6 unless a later stable Expo 55 patch selects another version, then rerun alignment and native verification |
| DEV-03 | Open acceptance gap | `bun run typecheck` fails with 19 errors, identical on the reconstructed SDK 54 and SDK 55 states | The errors predate the migration and are outside migration scope | The migration introduced no observed typecheck regression, but Gate A and overall acceptance remain failed | Resolve the existing type errors in separate work, then rerun typecheck on the migration state |
| DEV-04 | Open evidence gap | Android debug and release builds cover x86_64 only and were not installed or launched | This host has no Android emulator or attached device | Compilation and autolinking are proven only for one ABI; Android runtime and device behavior are unproven | Build required release ABIs, install on an emulator or device, launch, inspect logs, and complete Android gates |
